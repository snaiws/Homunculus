# 지침(Directive) 프레임워크

## 개요

지침은 호문쿨루스의 행동 규칙을 정의하는 YAML 파일입니다.
단순한 규칙이 아니라 **배경(왜)**과 **근거(어떤 제약에서)**를 포함하여,
호문쿨루스가 규칙의 의도를 이해하고 상황에 맞게 판단할 수 있도록 합니다.

## 파일 구조

```yaml
metadata:
  name: "지침 이름"
  version: 1
  author: "system"        # system | user | self
  created: "2026-02-28"
  modified: "2026-02-28"
  background: |
    왜 이 지침이 필요한가.
    어떤 문제를 해결하는가.
  rationale: |
    어떤 제약조건이나 목표에서 비롯되었는가.
  constraints:
    - "절대 위반하면 안 되는 조건들"
    - "사용자 승인 없이 변경 불가"

# 실제 설정값 (지침마다 다름)
settings:
  key: value
```

## 카테고리별 가이드

| 카테고리 | 배경 질문 | 근거 질문 |
|----------|----------|----------|
| schedule | 왜 이 시간에 자/깨나? | 토큰 한도, 사용자 패턴, 비용 최적화 |
| resources | 왜 이 한도인가? | 서비스 요금, 디스크 크기, 무료티어 제약 |
| behavior | 왜 이런 성격/스타일인가? | 사용자 선호, 작업 특성, 문화적 맥락 |
| channels | 왜 이 채널이 우선인가? | 사용자 접근성, 응답 속도, 알림 전달률 |
| maintenance | 왜 이 정리가 필요한가? | 디스크 한도, 메모리 품질, 부팅 속도 |

## 자기 수정 규칙

호문쿨루스가 지침을 자기 수정할 때:

1. **배경/근거 읽기**: 수정 전 반드시 background/rationale 확인
2. **의도 보존**: 근거의 제약조건을 위반하지 않는 범위에서만 수정
3. **author 변경**: `author: "self"`로 변경, 수정 이유를 background에 추가
4. **constraints 준수**: constraints는 사용자 승인 없이 변경 불가

## 파일 위치

모든 지침 파일은 `config/directives/` 디렉토리에 위치합니다.
파일명 규칙: `{카테고리}-{이름}.yaml` (예: `schedule-sleep.yaml`)
